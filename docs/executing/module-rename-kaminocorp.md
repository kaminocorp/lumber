# Module Rename: `hejijunhao/lumber` → `kaminocorp/lumber`

**Goal:** Move the repository from the personal GitHub account (`hejijunhao`) to the Kamino Corp organisation (`kaminocorp`) and update every reference so the codebase builds, tests pass, and documentation points to the new home.

**Starting point:** The Go module was declared as `github.com/hejijunhao/lumber` in `go.mod`. Every internal import across 36 `.go` files referenced this path. Documentation (README, changelogs, blueprints) contained clone URLs, `go get` commands, and import examples using the old path. The git remote pointed to `https://github.com/hejijunhao/lumber.git`.

**Why now:** No external consumers depend on the old module path yet (pre-v1.0, not published to `proxy.golang.org`). Renaming after external adoption would require maintaining a redirect module at the old path indefinitely.

## Implementation Plan

### Section 1: Update `go.mod` Module Declaration

**File:** `go.mod`

The module declaration is the root of Go's import resolution. Every other import path derives from this line. Change it first — until this matches the new GitHub org, nothing else will compile.

```
# Before

# After
module github.com/kaminocorp/lumber
```

No other lines in `go.mod` change. The `require` block references third-party modules (`github.com/yalue/onnxruntime_go`) which are unaffected.

## Execution Order

The sections must be executed in this specific order due to dependencies:

```
Section 1: go.mod                     ← must be first (compilation root)
    ↓
Section 2: Go source imports          ← depends on Section 1
Section 3: Go test imports            ← depends on Section 1, independent of Section 2
    ↓
Section 6: go mod tidy + build + test ← depends on Sections 1-3
    ↓
Section 4: Documentation              ← independent of build, but do after verification
Section 5: Git remote                 ← do last, after repo transfer on GitHub
```

Sections 2 and 3 can be done simultaneously (they are independent of each other). Section 4 (docs) can be done at any point but is best done after verifying the build so you're not fixing docs while the code is broken.

## Execution Method

All 128 references use the exact same string prefix. The entire rename can be executed as a single `sed` or equivalent find-and-replace:

```bash
# Preview (dry run — shows what would change):
grep -r "hejijunhao/lumber" --include="*.go" --include="*.md" -l

# Execute across all Go files + go.mod:
find . -name "*.go" -o -name "go.mod" | xargs sed -i '' 's|github.com/hejijunhao/lumber|github.com/kaminocorp/lumber|g'

# Execute across documentation:
find ./docs -name "*.md" | xargs sed -i '' 's|hejijunhao/lumber|kaminocorp/lumber|g'

# README is at root:
sed -i '' 's|hejijunhao/lumber|kaminocorp/lumber|g' README.md
```

Note: the docs replacement uses `hejijunhao/lumber` (without `github.com/` prefix) to catch both `github.com/hejijunhao/lumber` import paths and `https://github.com/hejijunhao/lumber.git` clone URLs in a single pass.
